# FPGA-Based Adaptive Beamformer for 5G Networks

A beamforming receiver for a 5G NR base station, implemented across MATLAB (algorithm and fixed-point analysis), SystemVerilog (RTL), and Vivado (synthesis on Artix-7). Phase 1 delivers a verified non-adaptive delay-and-sum beamformer; Phase 2 is a MATLAB-only algorithmic feasibility study comparing LMS and RLS for a future adaptive hardware implementation.

**Status:** all phases complete. Timing closure (Phase 1.5) and full hardware implementation of LMS (future Phase 3) are open items. See [PLAN.md](PLAN.md) §10.

---

## Headline results

| Phase | Result |
|---|---|
| 1.1 floating-point | Output SINR 15.78 dB (+18.33 dB over single element) |
| 1.2 fixed-point | SINR degradation 0.00 dB; output SQNR 33.51 dB |
| 1.4 RTL co-simulation | 512 / 512 samples PASS within ±1 LSB |
| 1.5 synthesis | Artix-7 xc7a35t — 16 DSPs, 812 LUTs, 930 registers |
| 2.1 LMS | SINR 18.93 dB; converges in 15 samples (best μ = 0.001) |
| 2.2 RLS | SINR 18.90 dB; converges in 2 samples (best λ = 1.000) |
| 2.3 recommendation | LMS for hardware — 8× lower MACs, < 0.03 dB SINR penalty |

Full results, plots, and discussion: [`project_update.pdf`](project_update.pdf).

## Quickstart

### Prerequisites

| Tool | Version used | Purpose |
|---|---|---|
| MATLAB | R2024a or later, Signal Processing Toolbox | All simulations |
| Icarus Verilog (`iverilog`) | 11.0 or later | RTL co-simulation |
| Vivado | 2025.2 (lab PC only) | Phase 1.5 synthesis |
| pdflatex | TeX Live | Compile `project_update.tex` |

### Run everything

```bash
make sim      # all five MATLAB simulations
make rtl      # iverilog co-simulation (512/512 PASS)
make report   # recompile project_update.pdf
make all      # sim + rtl + report
```

### Run the MATLAB simulations individually

```bash
cd matlab
matlab -nodisplay -nosplash -batch "algo_sim"        # Phase 1.1
matlab -nodisplay -nosplash -batch "fixed_point_sim" # Phase 1.2 (regenerates vectors/)
matlab -nodisplay -nosplash -batch "lms_sim"         # Phase 2.1
matlab -nodisplay -nosplash -batch "rls_sim"         # Phase 2.2
matlab -nodisplay -nosplash -batch "comparison"      # Phase 2.3
```

Each script calls `signal_setup()` for a consistent signal environment (M = 8, seed 42).

### Run the RTL co-simulation

```bash
make rtl   # or manually:
iverilog -g2012 -o beamformer_sim rtl/*.sv && vvp beamformer_sim
```

Run from the repository root so `vectors/*.hex` resolves correctly. Expected output: `512 / 512 PASS`.

### Run Vivado synthesis (lab PC)

```bash
make synth   # or: cd synthesis && vivado -mode batch -source synth.tcl
```

Targets `xc7a35tcpg236-1` in out-of-context mode. See `synthesis/synth.tcl` for details.

## Reproducibility notes

- All MATLAB simulations call `signal_setup()` which seeds `rng(42)` and draws signals in a fixed order. Do not call `rng()` externally.
- Test vectors in `vectors/` are regenerated by `fixed_point_sim.m`. The committed vectors correspond to M = 8, θ_s = 30°, θ_i = −20°, SNR = 10 dB, SIR = 0 dB.
- Synthesis was run in out-of-context mode. Timing closure was not achieved — documented as future work in [PLAN.md §7.3](PLAN.md) and as deferred requirement REQ-P1-015 in [REQUIREMENTS.md](REQUIREMENTS.md).
